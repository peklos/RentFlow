# RentFlow - Система управления арендой недвижимости

## О проекте

RentFlow - это полнофункциональная система управления арендой коммерческой и жилой недвижимости. Проект разработан на заказ для компании, занимающейся сдачей недвижимости в аренду, и представляет собой комплексное решение для автоматизации бизнес-процессов.

**Роль в проекте:** Full-stack разработчик
- 50% кода: отладка, деплой, интеграция модулей, архитектурные решения
- 50% кода: разработка функциональности с использованием AI-ассистента для ускорения рутинных задач

## Технический стек

### Backend
- **Python 3.11+** - основной язык программирования
- **FastAPI** - современный асинхронный веб-фреймворк
- **SQLAlchemy 2.0** - ORM для работы с БД
- **PostgreSQL** - реляционная база данных
- **Alembic** - миграции базы данных
- **Pydantic** - валидация данных и схемы
- **Uvicorn** - ASGI сервер

### Frontend
- **Vue.js 3** - прогрессивный JavaScript-фреймворк
- **Vite** - быстрый инструмент сборки
- **Vue Router** - маршрутизация
- **Pinia** - управление состоянием
- **Axios** - HTTP-клиент

### DevOps
- **Docker** - контейнеризация приложений
- **Netlify** - деплой frontend
- **Railway/Render** - деплой backend

## Особенности реализации

### Backend
- RESTful API с четкой структурой эндпоинтов
- Автоматическая документация API (Swagger/ReDoc)
- Валидация данных через Pydantic схемы
- Обработка ошибок и исключений
- CORS middleware для работы с frontend
- Автоматическая инициализация тестовых данных
- Система миграций БД

### Frontend
- Компонентная архитектура
- Centralized state management (Pinia)
- Route guards для защиты маршрутов
- Lazy loading компонентов
- Responsive дизайн
- Axios interceptors для обработки запросов
- Environment-based конфигурация

### Безопасность
> **Примечание:** По требованию заказчика в проекте не реализована JWT-токенизация и хеширование паролей для упрощения системы на этапе MVP.

## Установка и запуск

### Требования
- Python 3.11+
- Node.js 18+
- PostgreSQL 14+
- Docker (опционально)

### Backend

```bash
cd backend

# Создание виртуального окружения
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Установка зависимостей
pip install -r requirements.txt

# Настройка .env
cp .env.example .env
# Отредактируйте .env и укажите DATABASE_URL

# Запуск
uvicorn main:app --reload
```

API будет доступно на `http://localhost:8000`
- Swagger UI: `http://localhost:8000/docs`
- ReDoc: `http://localhost:8000/redoc`

### Frontend

```bash
cd frontend

# Установка зависимостей
npm install

# Настройка .env
cp .env.example .env
# Отредактируйте .env и укажите VITE_API_URL

# Запуск dev сервера
npm run dev
```

Приложение будет доступно на `http://localhost:5173`

### Docker

```bash
# Запуск всего проекта
docker-compose up -d
```

## API Endpoints (примеры)

### Клиент
```
POST   /api/client/auth/register        # Регистрация
POST   /api/client/auth/login           # Вход
GET    /api/client/profile              # Профиль
GET    /api/client/properties           # Список объектов
POST   /api/client/applications         # Создать заявку
GET    /api/client/contracts            # Мои контракты
GET    /api/client/payments             # Мои платежи
POST   /api/client/reviews              # Оставить отзыв
```

### Администратор
```
POST   /api/employee/auth/login         # Вход сотрудника
GET    /api/admin/statistics            # Статистика
GET    /api/admin/properties            # Управление объектами
PUT    /api/admin/applications/{id}     # Обновить заявку
POST   /api/admin/verifications         # Верифицировать клиента
GET    /api/admin/contracts             # Все контракты
GET    /api/admin/clients               # Все клиенты
```
